# RESOURCES.md Format

`RESOURCES.md` lives at the workspace root. It is a curated, annotated list of learning resources — not a dump of everything that exists, but a considered guide to what's worth using and when.

## Template

```md
# Resources: {Topic}

## Knowledge
> Structured input — courses, references, explanations.

### {Resource Name}
**Link**: {URL or "paid/app"}
**Format**: {Audio / Video / Text / App / Community}
**Cost**: {Free / Paid / Freemium}
**Reach for this when**: {specific situation or need}
{1-3 sentence annotation. What does it cover? What's good about it? Any caveats?}

---

## Skills
> Practice resources — exercises, tools, drills.

{Same structure}

---

## Wisdom
> Real-world use — communities, native speakers, immersion.

{Same structure}
```

## Rules

- **Annotate everything.** A link without context is noise. Every resource needs a "reach for this when" line.
- **Curate ruthlessly.** 5 well-annotated resources beat 20 unannotated links. If you wouldn't actually recommend it, leave it out.
- **Distinguish formats.** Audio-first resources are not the same as text-based ones. Flag the format clearly.
- **Note the dialect.** For language learning: flag whether a resource covers the target dialect (e.g. Dhaka Bangla) or a different one.
- **Update when resources change.** Apps get worse. Sites go down. YouTube channels stop updating. Prune when needed.
- **Flag free vs. paid.** Learners should know what they're getting into before clicking.
